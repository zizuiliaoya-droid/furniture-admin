# 家具软装内部管理平台 — 需求分析

## 意图分析

- **用户请求**: 根据开发文档和需求文档完成项目，测试环境使用 Vercel+Railway 与 GitHub 联动自动部署，生产环境部署到客户 Synology NAS 私有云
- **请求类型**: 新项目（Greenfield）
- **范围**: 系统级 — 完整的前后端应用（8个功能模块）
- **复杂度**: 中高

## 功能需求

### 核心模块（按开发优先级排序）

#### 1. 用户认证与权限
- Token 认证（Django REST Framework TokenAuthentication）
- ADMIN/STAFF 两种角色，管理员可增删改查，员工只能查看+部分创建
- 管理员可创建/编辑/启用/禁用用户账号
- 系统启动时自动创建初始管理员账号（来自环境变量）

#### 2. 产品管理
- 产品 CRUD（名称、编号、产地、描述、最低售价）
- 产地分类：进口、国产、定制
- 三维度树形分类体系（按类型、按空间、按产地）
- 多分类关联（一个产品可关联多个分类）
- 多图上传、封面设置、排序、缩略图自动生成
- 多配置价格（不同尺寸/材质，独立指导价格+自定义JSON属性）
- 上架/下架控制
- **[优化] 全字段搜索**：搜索范围覆盖产品名称、编号、描述、配置名称、配置属性
- **[优化] 分类拖拽排序**：支持拖拽调整分类排列顺序，后端提供批量更新排序接口
- **[优化] 相关资源快捷链接**：产品详情页可快速跳转到设计资源、培训资料、资质文件
- **[优化] Excel 批量导入**：支持通过 Excel 文件批量导入产品，提供标准模板下载

#### 3. 产品图册
- 卡片形式展示所有上架产品（封面图、名称、编号）
- 左侧树形分类导航，点击筛选对应分类下的产品
- 按名称/编号搜索
- 分页加载

#### 4. 客户案例
- 案例 CRUD（标题、描述、行业分类）
- **[优化] 行业分类**：改为办公空间8大细分行业+其他（科技/互联网、金融/保险/财税、地产/建筑/设计院、教育培训、医疗/大健康、广告/文创/传媒、制造/实业/工厂、政府/国企/事业单位、其他）
- 多图上传、封面设置、缩略图
- 关联产品，详情页展示关联产品信息
- 列表页按行业筛选

#### 5. 内部文档
- 三种文档类型：设计资源（DESIGN）、培训资料（TRAINING）、资质文件（CERTIFICATE）
- 树形文件夹结构，支持创建文件夹
- 文件上传（最大50MB）、下载
- 按文件名搜索
- 标签系统（添加标签、按标签筛选）
- **[优化] 文件夹删除**：文件夹树增加删除按钮，支持删除空文件夹
- **[优化] 在线预览**：支持图片、PDF、音频、视频的在线预览/播放
- **[优化] 音频播放**：培训资料支持音频文件在线播放
- **[优化] 分页加载**：文档列表分页加载，避免大量图片卡顿

#### 6. 报价方案
- 报价单 CRUD（标题、客户名称、备注、条款）
- 从产品库搜索添加产品到报价单，支持选择配置
- 价格计算：单价 × 数量 × (1 - 折扣/100)，自动计算小计和总金额
- 状态管理：草稿 → 已发送 → 已确认 → 已取消
- 一键复制报价单（含所有明细）
- PDF 导出
- 按标题/客户名搜索，按状态筛选

#### 7. 分享功能
- 可分享产品、案例、报价单、产品图册
- 访问控制：密码保护、过期时间、最大访问次数
- 启用/禁用分享链接
- 外部客户通过链接直接访问，无需登录
- 访问记录（时间、IP、浏览器信息）

#### 8. 全局搜索
- 跨模块搜索（产品、案例、文档、报价单）
- 顶部搜索栏实时搜索，下拉展示结果
- 点击结果直接跳转到对应详情页

## 非功能需求

- **数据库**: 仅 PostgreSQL 16（不支持 MySQL）
- **存储**: 仅本地磁盘存储（不使用云对象存储）
- **UI**: Ant Design 5.x 包豪斯风格定制主题
- **部署**: 测试环境 Vercel+Railway（GitHub 自动部署），生产环境 NAS Docker Compose

## UI 设计规范

### 设计风格：包豪斯（Bauhaus）

以包豪斯设计哲学为核心，强调几何形态、功能主义和极简美学，通过精致的细节营造高级感。

### 色彩体系
- **主色**: 纯黑 `#000000` + 纯白 `#FFFFFF`（包豪斯经典对比）
- **强调色**: 包豪斯三原色 — 红 `#E63946`、蓝 `#1D3557`、黄 `#F4A261`（克制使用，仅用于关键操作和状态标识）
- **中性色**: 灰阶梯度 `#F8F9FA` / `#E9ECEF` / `#6C757D` / `#343A40`
- **背景色**: 暖白 `#FAFAF8`（避免纯白的冰冷感）

### 字体方案
- **标题字体**: `"DM Sans", "Noto Sans SC", sans-serif`（几何感强，契合包豪斯风格）
- **正文字体**: `"Inter", "Noto Sans SC", sans-serif`（高可读性，现代感）
- **数字/代码**: `"JetBrains Mono", monospace`（报价单金额等场景）
- 通过 Google Fonts 引入，确保中英文混排效果

### 微交互设计
- **按钮交互**: 点击时轻微缩放（scale 0.97）+ 短暂模糊过渡（150ms ease-out）
- **卡片悬浮**: hover 时微上移（translateY -2px）+ 阴影扩散
- **页面切换**: 淡入淡出过渡（fade + slide，200ms）
- **列表项**: 交错动画入场（stagger animation）
- **加载状态**: 骨架屏（Skeleton）替代传统 loading spinner

### 微阴影系统（Soft Shadow）
- **Level 1（卡片）**: `0 2px 20px rgba(0, 0, 0, 0.04)`
- **Level 2（悬浮）**: `0 4px 30px rgba(0, 0, 0, 0.06)`
- **Level 3（弹窗）**: `0 8px 40px rgba(0, 0, 0, 0.08)`
- 抛弃深色硬阴影，使用超长弥散半径的柔和阴影，让界面元素像浮在光影中

### 布局原则
- **网格系统**: 严格的几何网格对齐，间距使用 8px 倍数（8/16/24/32/48）
- **留白**: 大量留白，内容区域不超过页面 70% 宽度
- **边框**: 极细线条（1px `#E9ECEF`）或无边框，依靠阴影和间距区分层级
- **圆角**: 统一小圆角 4px（包豪斯偏好几何硬朗感，避免大圆角）

## 关键技术决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 测试部署 | Vercel + Railway | 简化管理，原生支持 GitHub 自动部署 |
| 生产部署 | Synology NAS Docker | 客户私有云环境 |
| 数据库 | 仅 PostgreSQL | 简化开发，MySQL 后续按需添加 |
| 文件存储 | 仅本地磁盘 | 不考虑阿里云 OSS |
| CI/CD | GitHub → 自动部署 | 推送即部署，零手动操作 |
| UI 风格 | Ant Design 包豪斯定制主题 | 几何感、功能主义、高级感微交互 |
| 开发顺序 | 按文档顺序 | 认证→产品→图册→案例→文档→报价→分享→搜索 |
