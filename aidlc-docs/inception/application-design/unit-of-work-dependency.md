# 工作单元依赖矩阵

## 依赖关系

| 单元 | 前置依赖 | 可并行 | 说明 |
|------|----------|--------|------|
| Unit 1: 项目基础+认证 | 无 | — | 所有单元的基础 |
| Unit 2: 产品管理 | Unit 1 | — | 依赖认证和公共工具 |
| Unit 3: 产品图册 | Unit 2 | Unit 4, 5 | 依赖产品数据 |
| Unit 4: 客户案例 | Unit 2 | Unit 3, 5 | 依赖产品（关联产品） |
| Unit 5: 内部文档 | Unit 1 | Unit 3, 4 | 仅依赖认证和文件存储 |
| Unit 6: 报价方案 | Unit 2 | — | 依赖产品（报价明细关联产品） |
| Unit 7: 分享功能 | Unit 2, 4, 6 | — | 依赖产品、案例、报价（分享内容源） |
| Unit 8: 全局搜索+收尾 | Unit 1-7 | — | 依赖所有模块（跨模块搜索） |

## 开发顺序（关键路径）

```
Unit 1 (基础+认证)
  |
  v
Unit 2 (产品管理)
  |
  +---> Unit 3 (图册)     \
  |                         |
  +---> Unit 4 (案例)       +---> Unit 7 (分享)
  |                         |
  +---> Unit 5 (文档) *    /
  |
  +---> Unit 6 (报价) ----/
                                    |
                                    v
                              Unit 8 (搜索+收尾)
```

* Unit 5 仅依赖 Unit 1，可与 Unit 2 并行开发，但按用户指定顺序排在 Unit 4 之后。

## 并行开发机会

如果需要加速开发，以下单元可以并行：
- Unit 3 + Unit 4 + Unit 5（在 Unit 2 完成后）
- Unit 3 + Unit 6（在 Unit 2 完成后）

## 集成测试检查点

| 检查点 | 时机 | 验证内容 |
|--------|------|----------|
| CP1 | Unit 1 完成后 | 登录/登出、Token 认证、权限控制、Docker 启动 |
| CP2 | Unit 2 完成后 | 产品 CRUD、图片上传缩略图、分类树、Excel 导入 |
| CP3 | Unit 3-5 完成后 | 图册浏览、案例关联产品、文档上传预览 |
| CP4 | Unit 6 完成后 | 报价单创建、明细计算、PDF 导出 |
| CP5 | Unit 7 完成后 | 分享链接生成、密码验证、公开访问 |
| CP6 | Unit 8 完成后 | 全局搜索、整体联调、部署验证 |
